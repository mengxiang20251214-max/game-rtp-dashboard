# 后台「智能运营配置」使用说明

> 入口：登录后台 → **概览** 页底部「智能运营配置」模块。

一键根据当前所有游戏，自动生成一套有真实感的运营展示数据（RTP、玩家人数、投注金额、HOT / PUPULER / NEW 等）。常用于演示或快速铺一套合理的展示数据，省去逐个手动配置。

## ⚠️ 数据性质

这些是**「真实感模拟运营数据」**，由系统按运营权重模型自动生成，**不是任何真实第三方平台的真实数据**。仅用于前台展示效果，请勿对外宣称为真实平台数据。

生成是**确定性的**：同一天多次「生成预览」结果一致，换一天才会变化，不会每次刷新都乱跳。

## 三个操作

### 1. 生成预览
点「生成预览」。系统计算出新配置并展示摘要，**此时不会写入数据库**。预览内容包括：

- 总游戏数、HOT / PUPULER / NEW 数量
- RTP 最小 / 最大 / 平均
- 玩家人数、投注金额的最小 / 最大 / 总数
- 各分类数量分布
- 前 10 个变化最大的游戏「前 → 后」对比

确认效果满意后再进行下一步。**没有生成预览时，「确认应用」按钮不可点。**

### 2. 确认应用
点「确认应用」，会弹出二次确认。确认后系统会：

1. **先自动备份**当前全部游戏的展示字段（用于随时回滚）；
2. 再把预览的新配置批量写入数据库；
3. 成功后页面刷新，前台即生效。

> 备份与写入在同一个事务里完成，**先备份再更新**，应用过程中失败不会留下半套数据。

### 3. 恢复上次备份（回滚）
点「恢复上次备份」，二次确认后，把游戏的展示字段恢复到**最近一次应用前**的状态。按钮上方会显示上次备份的时间与游戏数；没有备份时按钮不可点。

## 不会改动的内容

智能运营配置**只生成展示运营字段**（RTP、目标 RTP、玩家数、投注、HOT 标记），**不会改动**：

- 游戏名称、图片、厂商、分类等基础信息
- 排名算法与接口字段结构
- 前台视觉主题

> 排名、热度（HOT / PUPULER）、NEW 标记仍由系统在读取时实时计算；前台金色焦点卡始终只出现在排名第 1 名。

## 权限

三个操作都需要后台管理员登录，未登录调用接口会返回 401。
